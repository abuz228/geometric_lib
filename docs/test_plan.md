# План тестирования для GEOMETRIC_LIB

## 1. Цели и задачи тестирования

**Цель:** Обеспечить корректность работы функций вычисления площади и периметра геометрических фигур.
**Задачи:**

- Проверить правильность математических вычислений
- Убедиться в корректной обработке граничных случаев
- Проверить обработку некорректных входных данных
- Автоматизировать процесс проверки

## 2. Описание тестируемого продукта

Проект "GEOMETRIC_LIB" представляет собой библиотеку для вычисления площади и периметра четырёх геометрических фигур:

- Круг (`circle.py`)
- Квадрат (`square.py`)
- Прямоугольник (`rectangle.py`)
- Треугольник (`triangle.py`)

Каждый модуль содержит:

- Функцию `area()` для вычисления площади
- Функцию `perimeter()` для вычисления периметра
- Подробную документацию на русском языке
- Обработку ошибочных данных (возврат -1)

## 3. Область тестирования

**Тестируются все функции из всех модулей:**

- `circle.area(r)`, `circle.perimeter(r)`
- `square.area(a)`, `square.perimeter(a)`
- `rectangle.area(a, b)`, `rectangle.perimeter(a, b)`
- `triangle.area(a, h)`, `triangle.perimeter(a, b, c)`

## 4. Стратегия тестирования

**Тип тестирования:** Модульное (Unit) тестирование
**Инструмент:** стандартный модуль Python `unittest`
**Подход:** Позитивное и негативное тестирование

**Тестовые сценарии включают:**

- Нормальные значения (целые и дробные числа)
- Граничные значения (нули)
- Некорректные данные (отрицательные числа)
- Ошибочные типы данных (строки, списки)
- Большие и малые значения чисел

## 5. Критерии приемки

- Все тесты должны проходить без ошибок
- Для корректных входных данных возвращается правильный результат
- Для некорректных данных возвращается -1
- Погрешность вычислений с плавающей точкой не превышает 1e-10

## 6. Ожидаемые результаты

- 100% покрытие тестируемых функций
- Отчет о прохождении всех тестов
- Подтверждение корректности математических вычислений
- Документация по запуску тестов
